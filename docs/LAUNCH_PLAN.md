# Launch Plan (Beta)

## Phases

### Phase 1: Internal Testing
- Closed access (whitelist 10-20 users)
- E2E check on 2-3 internal accounts

### Phase 2: Beta Launch
- Enable beta limits (1 chat / 5 min)
- Onboard first wave
- Daily incident review and adjustments

### Phase 3: Soft Launch
- One week with closed beta
- Monitor and iterate

## Growth Strategy

- 90% growth effort goes to web release
- Open access from day one
- Early adopter achievement for first 1000 users

- Daily limit: TBD
